## 步驟 0：環境準備

1. 用 Android Studio 開啟專案根目錄。
2. 等 Gradle Sync 完成（右下角進度條跑完、無紅字錯誤）。
3. 要跑要求 1、3 前，先開一台模擬器：
   - 上方工具列 Device Manager → 選一台 AVD → 按 ▶ 啟動，
   - 或接上實體 Android 手機（需開啟 USB 偵錯）。

---

## 步驟 1：執行要求 2（最快，不用模擬器）

**方法 A — Android Studio（建議）**
1. 開啟 `app/src/test/java/com/example/inventory/ItemDetailsViewModelTest.kt`
2. 右鍵類別名稱 `ItemDetailsViewModelTest` → **Run 'ItemDetailsViewModelTest'**
3. 確認三個方法都 ✅ 綠勾：
   - `reduceQuantityByOne_whenQuantityGreaterThanZero_updatesItemWithDecrementedQuantity`
   - `reduceQuantityByOne_whenQuantityIsZero_doesNotCallUpdateItem`
   - `deleteItem_callsRepositoryDeleteItem`

**方法 B — 指令列**
```bash
./gradlew test --tests "com.example.inventory.ItemDetailsViewModelTest"
```

---

## 步驟 2：執行要求 1 & 3（需要模擬器 / 實機）

**方法 A — Android Studio（建議）**
1. 確認模擬器已啟動。
2. 開啟 `app/src/androidTest/java/com/example/inventory/ItemDaoTest.kt`
3. 跑整個類別：右鍵 `ItemDaoTest` → **Run 'ItemDaoTest'**（會跑全部 7 個方法）
4. 或單獨跑某一題：
   - 要求 1：右鍵 `insertDuplicateId_keepsOriginalItem` → Run
   - 要求 3：右鍵 `insertStudentIdItem_verifyStoredInDatabase` → Run

**方法 B — 指令列**
```bash
# 跑整個 ItemDaoTest
./gradlew connectedAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=com.example.inventory.ItemDaoTest
```

## 步驟 3：要求 3 截圖（作業必須）

作業規定第三題要附截圖：

1. 在 Android Studio 單獨右鍵 `insertStudentIdItem_verifyStoredInDatabase` → **Run**
2. 等 Run 面板顯示綠勾，畫面會出現：
   ```
   com.example.inventory.ItemDaoTest > insertStudentIdItem_verifyStoredInDatabase PASSED
   ```
3. 截下整個 Run 面板（含測試名稱與 PASSED 字樣）。
4. 把截圖貼進 `HW3_submission.md` 第 5 節。
